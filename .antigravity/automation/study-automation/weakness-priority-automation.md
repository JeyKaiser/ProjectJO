# Weakness Priority Automation

## Objetivo
Priorizar automáticamente debilidades críticas.

## Señales críticas
- errores repetitivos,
- confusión persistente,
- debugging fallido,
- reasoning superficial.

## Reglas
- una debilidad crítica bloquea progresión,
- debilidades arquitectónicas tienen alta prioridad,
- debilidades fundamentales superan optimizaciones avanzadas.

## Objetivo pedagógico
Resolver causas raíz antes de escalar dificultad.
