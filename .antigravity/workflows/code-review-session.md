# Code Review Session Workflow

## Paso 1: Selección del archivo
Elegir el componente, página o módulo a revisar.

## Paso 2: Lectura estructural
Entender responsabilidades, dependencias y flujo de datos.

## Paso 3: Diagnóstico
Detectar:
- smells,
- acoplamientos,
- duplicación,
- complejidad innecesaria,
- riesgos de rendimiento.

## Paso 4: Evaluación
Clasificar hallazgos por severidad.

## Paso 5: Recomendación
Proponer mejoras concretas y justificadas.

## Paso 6: Seguimiento
Registrar qué hallazgos deben priorizarse.
