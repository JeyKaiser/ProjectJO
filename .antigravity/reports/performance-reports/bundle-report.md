# Bundle Report

## Objetivo
Evaluar tamaño, composición y oportunidades de optimización del bundle.

## Criterios
- vistas pesadas
- imports innecesarios
- componentes poco usados
- posibilidad de lazy loading
- separación por rutas

## Riesgos
- bundle inicial grande
- tiempo de carga alto
- navegación lenta
- dependencia de módulos pesados

## Mejores prácticas
- code splitting
- carga diferida
- imports selectivos
- evitar acoplamientos innecesarios

## Salida esperada
- diagnóstico del bundle
- puntos de carga pesada
- oportunidades de división
- plan de optimización
