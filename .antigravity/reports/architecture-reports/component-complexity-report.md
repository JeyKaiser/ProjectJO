# Component Complexity Report

## Objetivo
Medir la complejidad estructural de los componentes del proyecto.

## Criterios
- cantidad de responsabilidades
- cantidad de estados
- cantidad de efectos
- cantidad de props
- tamaño del JSX
- claridad de intención

## Riesgos
- god components
- componentes demasiado largos
- mezcla de lógica con presentación
- dificultad para testear
- alta fragilidad ante cambios

## Señales de buena salud
- componentes pequeños
- composición clara
- lógica extraída
- props mínimas
- render predecible

## Salida esperada
- ranking de complejidad
- nivel de riesgo
- sugerencias de extracción
- propuesta de reestructuración
