# Topic Priority System

## Objetivo
Determinar qué temas deben estudiarse primero.

## Prioridad alta
- render lifecycle,
- state architecture,
- separación de responsabilidades,
- debugging.

## Prioridad media
- memoización,
- routing avanzado,
- arquitectura CSS.

## Prioridad baja
- micro optimizaciones,
- abstracciones avanzadas prematuras.

## Regla
Los temas críticos del proyecto siempre tienen prioridad.
