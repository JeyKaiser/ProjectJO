# Challenge Recommendation System

## Objetivo
Elegir retos ideales según perfil técnico.

## Variables
- scores,
- debilidades,
- confianza,
- roadmap.

## Ejemplos
- bajo performance score → render analysis,
- bajo architecture score → separación de responsabilidades,
- debugging débil → bugs contextuales.

## Regla
Cada reto debe tener propósito pedagógico claro.
