# Challenge Generation Rules

## Objetivo
Generar retos técnicos personalizados según:
- nivel actual,
- errores recurrentes,
- componentes del proyecto,
- debilidades detectadas,
- progreso técnico.

## Variables de entrada
- confidence-levels,
- weaknesses,
- completed-topics,
- performance reviews,
- architecture analysis.

## Tipos de retos
- debugging,
- performance,
- refactoring,
- arquitectura,
- state management,
- routing,
- CSS scaling.

## Reglas
- nunca generar retos demasiado fáciles,
- evitar ejercicios desconectados del proyecto,
- priorizar problemas reales del repositorio,
- aumentar dificultad progresivamente.

## Objetivo pedagógico
Entrenamiento contextual y adaptativo.
