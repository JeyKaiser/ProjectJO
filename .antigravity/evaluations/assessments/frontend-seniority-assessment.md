# Frontend Seniority Assessment

## Junior
- entiende componentes,
- usa hooks básicos,
- comprende props y estado,
- puede modificar UI simple.

## Mid-Level
- comprende render lifecycle,
- identifica malas prácticas,
- maneja routing y estado,
- puede refactorizar componentes.

## Senior
- piensa en escalabilidad,
- evalúa tradeoffs,
- diseña arquitectura,
- optimiza performance,
- modela dominio,
- separa responsabilidades correctamente.

## Evaluación
La seniority NO depende de:
- cantidad de hooks conocidos,
- complejidad visual,
- cantidad de código escrito.

Depende de:
- criterio técnico,
- capacidad de diseño,
- mantenibilidad,
- claridad arquitectónica.
