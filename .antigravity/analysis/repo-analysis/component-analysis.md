# Component Analysis

## Objetivo
Analizar componentes React para detectar complejidad, acoplamiento y problemas de composición.

## Elementos a evaluar
- tamaño,
- responsabilidades,
- props,
- lógica interna,
- estado,
- reutilización,
- claridad.

## Riesgos
- God Components,
- lógica excesiva,
- JSX demasiado largo,
- demasiados estados,
- efectos complejos,
- componentes difíciles de testear.

## Señales positivas
- responsabilidades claras,
- separación UI/lógica,
- composición limpia,
- hooks reutilizables,
- render predecible.

## Preguntas de análisis
- ¿El componente hace demasiadas cosas?
- ¿La lógica puede extraerse?
- ¿Existen responsabilidades mezcladas?
- ¿La composición puede mejorar?

## Objetivo pedagógico
Aprender a evaluar componentes como arquitecto frontend senior.
