# Domain Model Analysis

## Objetivo
Evaluar cómo el dominio del negocio está representado en el frontend.

## Evaluar
- temperaturas,
- fases,
- referencias,
- reglas,
- transiciones.

## Riesgos
- lógica mezclada con UI,
- reglas duplicadas,
- inconsistencias visuales,
- estados ambiguos.

## Señales positivas
- dominio centralizado,
- semántica consistente,
- reglas explícitas,
- estados previsibles.

## Objetivo pedagógico
Aprender arquitectura orientada al dominio.
